## F-04. [통합 검색 REST API 엔드포인트 구축]

- **목적:** 프론트엔드에서 보낸 요청을 받아 내부의 각 AI 모델과 DB를 제어하고 결과를 반환하는 중앙 서버 역할을 수행함.
- **입력:** Client의 HTTP POST 요청 (JSON 형식의 검색어 `query`)
- **출력:** 요청 처리 상태, 의도 분류 결과, 유사도 검색 결과를 모두 포함한 통합 JSON 응답

- **정상 흐름 (사용자 관점):**
  1. 클라이언트가 `/api/search` 엔드포인트로 검색어 데이터를 전송한다.
  2. 서버(FastAPI)가 요청을 파싱하고 의도 분류 모듈을 호출한다.
  3. 분류 결과와 검색어를 로깅 모듈(SQLite)에 전달하여 기록한다.
  4. 검색 의도일 경우 FAISS 서비스 모듈을 호출하여 결과를 취합한 뒤, 클라이언트로 `200 OK` 응답과 함께 결과 JSON을 반환한다.

- **예외/실패 처리:**
  - 잘못된 JSON 포맷이나 필수 파라미터(`query`)가 누락된 경우 `422 Unprocessable Entity` 에러 반환.
  - 내부 AI 모델이나 DB 통신 중 오류 발생 시 `500 Internal Server Error` 반환 후 에러 로그 기록.

- **완료 기준 (DoD):** Postman 또는 프론트엔드 환경에서 검색 API를 호출했을 때, 모든 내부 로직이 순차적으로 문제없이 실행되고 최종 JSON 데이터가 정확히 반환되어야 함.
